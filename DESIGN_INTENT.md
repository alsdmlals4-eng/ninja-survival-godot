# DESIGN_INTENT

## 기획 의도

`닌자 서바이벌`은 복잡한 조작보다 빠른 전투 피드백, 자동 공격의 연속 처치감, 짧은 성장 선택의 재미를 우선한다.

## 플레이어에게 주고 싶은 감정

- 빠르게 강해지는 감각
- 적 무리를 베어 넘기는 압박과 해소
- 닌자다운 회피/오의 사용 타이밍의 긴장감
- 빌드가 맞물릴 때 생기는 연속 도파민

## 핵심 선택 구조

초기 Godot MVP에서는 선택 구조를 최소화한다. 먼저 전투의 기본 손맛을 검증한 뒤, 성장 선택과 문파/인법 시너지를 추가한다.

## 자동/수동 조작 구분

- 자동: 기본 공격, 전투 인법, 장비 효과, 소환/장판
- 수동: 이동, 보법, 오의
- 휴식 조작: 가방 정리, 조합, 판매, 강화, 미션 선택

## 벤치마킹 반영 원칙

Vampire Survivors류의 핵심은 많은 기능보다 반복 전투의 즉각적인 보상감이다. 다른 작품의 시스템을 그대로 복사하지 않고, 닌자 세계관의 속도감과 인법 성장에 맞춰 변환한다.

## 금지 방향

- Unity 구조를 Godot에 그대로 복사하지 않는다.
- 첫 MVP에 모든 시스템을 넣지 않는다.
- 자동 공격 게임을 복잡한 수동 액션 게임으로 바꾸지 않는다.
- 닌자/인법 판타지를 흐리는 일반 RPG식 복잡한 전투 수치를 먼저 늘리지 않는다.
