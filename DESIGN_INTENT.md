# DESIGN_INTENT

## 기획 의도

`닌자 서바이벌`은 복잡한 조작보다 빠른 전투 피드백, 자동 인법의 연속 처치감, 휴식 구간의 빌드 설계, 백팩 배치 시너지의 반복 재미를 우선한다.

## 플레이어에게 주고 싶은 감정

- 빠르게 강해지는 감각
- 적 무리를 쓸어내는 압박과 해소
- 처치 콤보와 스타일리쉬 점수로 생기는 즉시 도파민
- 유파별 고유 규칙이 전투 리듬을 바꾸는 감각
- 스테이지 결과를 보고 다음에 무엇을 키울지 판단하는 설계감
- 백팩 배치와 조합이 다음 전투를 바꾸는 만족감
- 운명 선택으로 한 판의 규칙이 달라지는 긴장감
- 최종 결과에서 이번 판의 닌자 랭크, MVP, 결말문이 기억되는 감각

## 핵심 선택 구조

현재 기획 MVP는 단순 전투만 검증하지 않는다. 전투 DDD, 4유파 감정 차이, 스테이지 결과 판단, 휴식 구간, 백팩 기초, 운명 선택, 최종 결과가 서로 연결되는지 검증한다.

단, 각 기능은 얕게 구현한다. 전체 스킬풀, 복잡한 백팩, 완전한 상점 경제, 정교한 엔딩 분기는 후속으로 둔다.

## 자동/수동 조작 구분

- 자동: 기본 공격, 전투 인법, 장비 효과, 소환/장판
- 수동: 이동, 보법, 오의
- 휴식 조작: 가방 정리, 조합, 판매, 강화, 운명/미션 선택

## 전투 DDD 방향

전투 중 가장 중요한 즉시 보상은 처치 콤보와 보상 흡수다.

MVP에서는 성능 보상보다 표현 중심으로 구현한다.

- 처치 콤보 표시
- 스타일리쉬 점수 누적
- 경험치/골드/보상 흡수 연출
- 콤보 실패 페널티 없음
- 결과 화면에서 스타일리쉬 점수와 랭크로 반영

## 스테이지 결과 방향

5/10/15분 보스 처치 후에는 중간 결과 화면을 보여준다.

중간 결과는 최종 정산이 아니라 다음 휴식에서 무엇을 강화할지 판단하게 만드는 화면이다.

표시 지표:

- 딜량 1위
- 힐량 1위
- 방어량 1위
- 상태이상 부여 1위
- 콤보/처치 1위
- 최대 콤보
- 주요 획득물
- 추천 성장 힌트 1~2개

## 휴식 구간 방향

휴식은 길고 피곤한 관리 화면이 아니라 짧은 빌드 설계 구간이다.

기본 흐름:

```text
전투 결과 요약
→ 전리품 확인
→ 현재 백팩 확인
→ 배치/정리
→ 조합 가능 확인
→ 상점/강화
→ 운명/미션 선택
→ 다음 전투 예고
```

## 벤치마킹 반영 원칙

Vampire Survivors류의 핵심은 많은 기능보다 반복 전투의 즉각적인 보상감이다.

Brotato류의 강점은 짧은 전투 후 빠른 선택 리듬이다.

Backpack Battles/Backpack Hero류의 강점은 배치와 조합이 전투 성과를 바꾸는 점이지만, 정보 과부하와 반복 피로가 위험이다.

따라서 닌자 서바이벌은 다음 원칙을 따른다.

- 전투 중에는 싸우고 얻는다.
- 휴식 중에는 읽고 정리하고 선택한다.
- 결과 화면은 정산표가 아니라 다음 선택의 근거다.
- 백팩은 최종적으로 깊게 가지만 MVP에서는 칸 제한과 인접 시너지만 검증한다.

## 금지 방향

- Unity 구조를 Godot에 그대로 복사하지 않는다.
- 자동 공격 게임을 복잡한 수동 액션 게임으로 바꾸지 않는다.
- 첫 구현 Goal 하나에 현재 기획 MVP 전체를 넣지 않는다.
- 전투 중 백팩 편집을 허용하지 않는다.
- 콤보 실패 페널티를 먼저 넣지 않는다.
- 추천 성장 힌트를 강제 선택으로 만들지 않는다.
- 닌자/인법 판타지를 흐리는 일반 RPG식 복잡한 전투 수치를 먼저 늘리지 않는다.
