# DOCUMENTATION_MAP

## 목적

이 문서는 `ninja-survival-godot` 저장소의 문서 역할을 구분한다.

## 핵심 문서

| 문서 | 역할 | 상태 |
|---|---|---|
| `AGENTS.md` | AI/Codex 최상위 작업 규칙 | 초기 작성 |
| `README.md` | 저장소 소개와 현재 상태 | 초기 작성 |
| `PROJECT_BRIEF.md` | 프로젝트 한 줄 설명, 장르, 핵심 경험 | 초기 작성 |
| `DESIGN_INTENT.md` | 기획 의도와 금지 방향 | 초기 작성 |
| `MVP_ROADMAP.md` | MVP 순서와 완료 기준 | 초기 작성 |
| `TEST_CHECKLIST.md` | 실행/검수 체크리스트 | 초기 작성 |
| `docs/BASE_RULES_VERSION.md` | Base 공용 규칙 동기화 기준 | 초기 작성 |
| `docs/UNITY_MIGRATION_AUDIT.md` | Unity 원본 분석 결과 | Unity ZIP 확인 후 갱신 필요 |
| `docs/GODOT_PORT_PLAN.md` | Godot 전환 계획 | 초기 작성 후 Unity 분석 반영 필요 |
| `docs/SYSTEM_MAP.md` | 시스템/파일/노드 매핑 | Unity 분석 후 갱신 필요 |
| `docs/CODEX_GOAL_MVP_001.md` | Codex MVP-001 실행 지시문 | 초기 작성 |

## 작업 기준

- Unity 원본 분석 전에는 구현 완료를 추정하지 않는다.
- Unity 코드는 참고 자료로만 사용한다.
- Godot 구현 문서는 Godot/GDScript 용어로 작성한다.
- Base로 승격할 내용과 프로젝트 전용 내용을 구분한다.
